# 🚗 CJUParking — AI 기반 실시간 주차장 빈자리 인식 시스템

> 주차장 내 CCTV 영상을 기반으로 AI가 주차 공간의 점유 여부를 인식하고,  
> 인식 결과를 서버로 전달하여 사용자에게 **실시간 주차장 빈자리 정보**를 제공하는 시스템

---

## 📌 프로젝트 개요
- 개발 기간: 2025.04 - 2026.01
- 개발 인원: 6명 (대학 졸업 팀 프로젝트)
- 목적: 실시간 주차 현황 동기화 및 상태 관리 백엔드 구현


본 프로젝트는 주차장 내 CCTV 영상 데이터를 활용하여  
AI가 주차 공간의 점유 상태를 자동으로 인식하고,  
해당 결과를 서버에서 처리하여 **실시간 빈자리 정보**를 제공하는 것을 목표로 합니다.  

AI가 인식한 결과를 서버에서 처리하여  
주차 공간 상태가 자연스럽게 갱신되도록 구현했습니다.

---

## 🧠 시스템 전체 흐름

CCTV 영상
-> AI 주차 빈자리 인식
-> 빈자리 인식 결과 (JSON)
-> Backend Server
-> DB 상태 갱신
-> 클라이언트 빈자리 조회


## ✨ 주요 기능

- CCTV 기반 AI 빈자리 인식 결과 수신
- 실시간 주차장 빈자리 현황 조회
- 빈자리 변경 시 클라이언트 반영


## 📡 CCTV → 서버 데이터 수신 방식

AI 시스템은 CCTV 영상을 분석하여 주차 공간의 점유 여부를 판단한 후,
REST API 방식으로 서버에 인식 결과를 전달합니다.

AI 서버
-> 주차 공간 인식 결과 생성
-> HTTP POST 요청 (JSON)
-> Spring Boot API 수신
-> 주차 공간 상태 검증 및 갱신


## 🛠 처리 방식

-> AI 인식 결과를 JSON 형태로 수신

-> 주차 공간의 여부로 기존 상태와 비교

-> 상태 변경이 있는 경우에만 DB 반영

AI 인식 결과는 오차 가능성이 있기 때문에  
서버에서는 이전 상태와 비교하여  
의미 있는 변경만 반영하도록 설계하였습니다.

## 🛠 시스템 구조

 - main
   - java
     - cju.parkinggo.parking
       - config        : 전역 설정 (CORS, Swagger 등)
       - controller    : API 요청/응답 처리
       - dto           : 요청/응답 DTO
       - entity        : JPA 엔티티
       - repository    : 데이터 접근 계층
       - service       : 비즈니스 로직
       - ParkinggoApplication
   - resources
     - static
     - application.properties

구조 설명
Controller - Service - Repository 계층 구조를 기반으로  
역할에 따라 패키지를 분리하여 유지보수성과 가독성을 고려했습니다.

AI 인식 결과는 Controller에서 수신한 뒤  
Service 계층에서 기존 상태와 비교하여  
의미 있는 변경만 DB에 반영하도록 처리합니다.
